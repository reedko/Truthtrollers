The highlight explains why the evaluation exists. Instead of showing only a score, the interface reveals the path of reasoning:

Case → selected case claim → matching source claims → underlying source → evaluation state

That makes the evidence flow legible at a glance, especially for first-time users.

For the real product interaction, I’d make the default state fairly calm, then on hover or click: unrelated nodes dim, the matching case-claim path glows gold, and any source that feeds multiple case claims gets a secondary green accent so users immediately understand that one source can participate in several reasoning branches.
